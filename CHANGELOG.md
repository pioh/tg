# Changelog

Все значимые изменения проекта документируются здесь.
Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование — [Semantic Versioning](https://semver.org/lang/ru/).

**Как обновиться:** в боте — `/version` (проверить) и `/update` (обновить; агент сам
перезапустится, если запущен как сервис). В терминале — `bun run tg update`
(= `git pull` + `bun install`), затем перезапусти сервис. Подробнее — раздел
«Обновление» в [README](README.md#обновление).

## [0.7.0] - 2026-06-25

Плавный UX мультитенанта по `spec/it6.txt`. Сделано воркфлоу-агентами + перекрёстное ревью
+ 5 проходов независимой проверки Codex (финал — без P0/P1/P2).

### Changed
- **BREAKING**: интерактивный MCP теперь требует ЯВНОГО выбора пользователя — инструмент
  `set_context(<имя>)` (раз за сессию) + `list_tenants`. Никаких env для выбора тенанта и
  никакого авто-выбора единственного — скрытые дефолты убраны (главное требование).
- **Убрана защита отправки** (`assertCanSend`) и весь модуль разрешений: отправка в любой
  чат свободна, дисциплина «пиши посторонним только по просьбе/правилу» — в `rules/50`.
  Побочно починилось создание бота через @BotFather (раньше его блокировал гейт).
- Имя тенанта строго валидируется в `tenantDir` (`[A-Za-z0-9_-]`, без `..`/слэшей) —
  закрыт path-traversal и скрытый фолбэк на legacy `data/`.

### Added
- Надёжный авто-формат сообщений бота: **Markdown → Telegram HTML** (parse_mode HTML).
  Больше не уходят сырыми из-за спецсимволов MarkdownV2 (`_`, `.`, `(`, `@` и т.п.).
- Авто-регистрация владельца у бота в КОДЕ: после `bot_set_token` сервис сам шлёт `/start`
  от аккаунта владельца и ждёт привязку — без ручного «нажми Start».

### Fixed
- `setBotToken` при смене токена (в т.ч. через `TG_BOT_TOKEN`) сбрасывает старые
  `botOwnerChatId`/`botGroupChatId`/offset (иначе `bot_status` врал, а `/start` терялся).
- `assertSafeFile` — по `realpath`, блокирует секреты (session/config/state/lock/
  permissions/.env) у ВСЕХ тенантов и legacy, включая подкинутые в `downloads/`.
- `listTenants` игнорирует папки с кривым именем (не роняют старт сервиса).
- lock пишется ПОСЛЕ бинда хаба; `hubCall` ретраит на ECONNREFUSED/401 (перечитав lock);
  `setup` убивает временный child по таймауту; CLI работает в явном контексте тенанта.
- `doctor` пер-тенантный и сам авто-мигрирует legacy `data/`.

### Removed
- Разрешения на отправку: MCP `permission_*`, бот-команды `/grant` `/revoke` `/permissions`,
  CLI `tg permissions`; модуль `src/lib/permissions.ts`.
- Авто-выбор единственного тенанта (везде — только явный выбор).
- MarkdownV2 как формат доставки (заменён на Telegram HTML).

## [0.6.0] - 2026-06-24

### Changed
- **BREAKING**: убран legacy-режим единой папки `data/`. Никакого «двойного режима» —
  всё работает ТОЛЬКО на тенантах (`tenants/<имя>`). Путь рабочей папки берётся из
  контекста тенанта или `TG_DATA_DIR`, скрытого дефолта `data/` больше нет.

### Added
- **Авто-миграция**: при апгрейде со старой версии папка `data/` автоматически
  переносится в `tenants/main` — это делают `doctor` и старт сервиса, без ручных команд.
- Агенту — инструкция самостоятельной миграции формата (`rules/15-updates.md`): при смене
  версии посмотреть `CHANGELOG`/`git log` и привести свою рабочую папку к новому формату;
  текущая версия tg теперь видна в bootstrap-контексте.
- CLI `status`/`qa`/`permissions` работают в контексте пользователя (имя первым
  аргументом или единственный — авто); `doctor` показывает каждого пользователя отдельно
  (вход, движок/модель, статус сервиса) и сам мигрирует legacy `data/`.

### Fixed (робастность мультитенанта)
- **Изоляция сбоя тенанта**: «битый»/незалогиненный пользователь (например
  `AUTH_KEY_UNREGISTERED`) больше НЕ роняет весь процесс и остальных. Ранняя валидация
  авторизации до захвата lock/порта, ленивый `getMe`, глобальный guard от
  `unhandledRejection`, очистка lock/клиента при неудачном старте тенанта.
- **Динамический порт хаба** у каждого пользователя: предпочитается `8765+index`, при
  занятости берётся любой свободный (порт пишется в lock, MCP-прокси читает его оттуда).
  Больше нет конфликтов портов с другими приложениями на машине — сервис стабилен сам.

## [0.5.1] - 2026-06-24

### Fixed
Багфиксы по итогам независимой верификации (Codex):
- Ответ агента больше не «утекает» в чужой чат: цель ответа хранится как FIFO на каждый
  ход (а не один общий `lastBotChatId`); текст и индикатор «печатает…» идут в чат именно
  текущего хода, события без адресата (мониторы) — владельцу.
- Watchdog-рестарт не теряет события: пришедшие в окно backoff копятся и переигрываются
  в новую сессию; при падении `inflight` всегда сбрасывается (typing не «висит»); при
  окончательной остановке сессии события не пушатся в мёртвую очередь.
- `/model` для Codex применяется пересозданием сессии (раньше «молча» оставалась старая).
- Команды в группе, адресованные другому боту (`/cmd@OtherBot`), игнорируются.
- `/update` не рапортует успех, если `bun install` упал (и не перезапускается).
- `setup` поднимает временный сервис на свободном порту (не конфликтует с уже запущенным
  сервисом другого пользователя на 8765).
- Монитор `mentions_me` срабатывает на реальное @-упоминание или ответ именно на ТВОЁ
  сообщение (а не на любой reply в группе).

## [0.5.0] - 2026-06-24

### Added
- **Мультитенантность**: один сервис обслуживает НЕСКОЛЬКО независимых пользователей.
  У каждого своя рабочая папка `tenants/<имя>` со своей сессией Telegram, ботом,
  памятью, правилами, мониторами и RPC-хабом. Изоляция — через контекст тенанта
  (AsyncLocalStorage); все пути/состояние указывают на папку конкретного тенанта.
- CLI управления тенантами: `tg tenant list | add <имя> | migrate-legacy <имя>`,
  `tg login <имя>`, `tg setup <имя>`. `setup` спрашивает имя и настраивает одного тенанта.
- **Семейный/групповой режим**: бот может работать в общей группе — команда `/here`
  назначает группу «ассистент-группой», и бот отвечает на ВСЕ сообщения разрешённых
  участников (нужно выключить privacy-mode у @BotFather). `/forgethere` — отключить.
  Несколько человек (например, семья) общаются с одним ботом в группе.
- Команды бота в группах распознаются с суффиксом `@botusername`.
- `doctor` показывает список тенантов.

### Changed
- Рабочая папка теперь у каждого тенанта своя (`tenants/<имя>`); прежняя единая `data/`
  поддерживается как legacy-тенант до миграции (`tg tenant migrate-legacy <имя>`).
- Lock/хаб — per-tenant (свой порт у каждого), что и обеспечивает «один сервис — много
  пользователей».

## [0.4.0] - 2026-06-24

### Added
- Команды бота `/help` и `/start` с описанием возможностей; меню команд в Telegram
  (`setMyCommands`) и описание бота — подсказки видны прямо в клиенте.
- Команды бота: `/monitors`, `/schedules`, `/permissions`, `/grant`, `/revoke`,
  `/users`, `/allowuser`, `/denyuser`, `/version`, `/update`, `/restart`.
- Allowlist: кто может писать боту (по умолчанию — только владелец). Если пишет
  посторонний — бот уведомляет владельца и предлагает `/allowuser <id>`.
- Индикатор «печатает…» в чате бота, пока агент работает; гаснет при простое и при
  падении агента (не «висит» вечно).
- Авто-доставка текста агента в Telegram (формат **MarkdownV2**, длинное режется само) —
  ответ человеку не требует явного вызова инструмента.
- Установка фонового сервиса нативно для платформы: `bun run tg install-service`,
  `uninstall-service`, `service-help` (Linux — `systemd --user`, macOS — `launchd`,
  Windows — инструкция/`schtasks`). Мастер `setup` предлагает установку автоматически.
- Само-обновление: периодическая проверка новой версии (git-теги), предложение в боте,
  `/update` + само-перезапуск под менеджером сервиса.
- Маскировка пароля 2FA звёздочками при входе (`bun run login`).
- `CHANGELOG.md` и версионирование (SemVer).

### Changed
- Агент работает с полными локальными правами (Claude — `bypassPermissions`,
  Codex — `danger-full-access`): все инструменты, включая Bash. tg — личный локальный
  инструмент; в Telegram остаётся код-гейт отправки.
- `/effort` валидируется под конкретный движок (Codex: `minimal…xhigh`, без `max`).
- Промпты усилены: текстовый вывод агента = сообщение человеку в Telegram.

### Fixed
- «Выключенный монитор» больше не оставляет висящего разрешения на отправку (снимается
  при выключении/удалении монитора) — «выключено» теперь действительно значит выключено.
- `service --once` корректно завершается (останавливается RPC-сервер) — годен для cron.
- Убрано шумное авто-сообщение «принял, разбираю…» и дубли промежуточных 💭-статусов.

### Removed
- `monitor_poll` / `schedule_poll` из MCP-инструментов (опрос двигает курсоры — это
  делает сам сервис; интерактивный агент больше не «ворует» события у живого сервиса).
- `bot_progress` (💭) как основной канал — заменён прямой доставкой текста агента.

## [0.3.0] - 2026-06-24

### Added
- Первый публичный релиз: ИИ-агент личного Telegram (Bun + mtcute + MCP), движки
  Claude Code / Codex, дисковая память (handoff/progress/qa/rules), мониторы,
  расписания, сервисный бот, RPC-хаб с bearer-токеном, lock единственного сервиса.
