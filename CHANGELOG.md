# Changelog

Все значимые изменения проекта документируются здесь.
Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование — [Semantic Versioning](https://semver.org/lang/ru/).

**Как обновиться:** в боте — `/version` (проверить) и `/update` (обновить; агент сам
перезапустится, если запущен как сервис). В терминале — `bun run tg update`
(= `git pull` + `bun install`), затем перезапусти сервис. Подробнее — раздел
«Обновление» в [README](README.md#обновление).

## [0.4.0] - 2026-06-24

### Added
- Команды бота `/help` и `/start` с описанием возможностей; меню команд в Telegram
  (`setMyCommands`) и описание бота — подсказки видны прямо в клиенте.
- Команды бота: `/monitors`, `/schedules`, `/permissions`, `/grant`, `/revoke`,
  `/users`, `/allowuser`, `/denyuser`, `/version`, `/update`, `/restart`.
- Allowlist: кто может писать боту (по умолчанию — только владелец). Если пишет
  посторонний — бот уведомляет владельца и предлагает `/allowuser <id>`.
- Индикатор «печатает…» в чате бота, пока агент работает; гаснет при простое и при
  падении агента (не «висит» вечно).
- Авто-доставка текста агента в Telegram (формат **MarkdownV2**, длинное режется само) —
  ответ человеку не требует явного вызова инструмента.
- Установка фонового сервиса нативно для платформы: `bun run tg install-service`,
  `uninstall-service`, `service-help` (Linux — `systemd --user`, macOS — `launchd`,
  Windows — инструкция/`schtasks`). Мастер `setup` предлагает установку автоматически.
- Само-обновление: периодическая проверка новой версии (git-теги), предложение в боте,
  `/update` + само-перезапуск под менеджером сервиса.
- Маскировка пароля 2FA звёздочками при входе (`bun run login`).
- `CHANGELOG.md` и версионирование (SemVer).

### Changed
- Агент работает с полными локальными правами (Claude — `bypassPermissions`,
  Codex — `danger-full-access`): все инструменты, включая Bash. tg — личный локальный
  инструмент; в Telegram остаётся код-гейт отправки.
- `/effort` валидируется под конкретный движок (Codex: `minimal…xhigh`, без `max`).
- Промпты усилены: текстовый вывод агента = сообщение человеку в Telegram.

### Fixed
- «Выключенный монитор» больше не оставляет висящего разрешения на отправку (снимается
  при выключении/удалении монитора) — «выключено» теперь действительно значит выключено.
- `service --once` корректно завершается (останавливается RPC-сервер) — годен для cron.
- Убрано шумное авто-сообщение «принял, разбираю…» и дубли промежуточных 💭-статусов.

### Removed
- `monitor_poll` / `schedule_poll` из MCP-инструментов (опрос двигает курсоры — это
  делает сам сервис; интерактивный агент больше не «ворует» события у живого сервиса).
- `bot_progress` (💭) как основной канал — заменён прямой доставкой текста агента.

## [0.3.0] - 2026-06-24

### Added
- Первый публичный релиз: ИИ-агент личного Telegram (Bun + mtcute + MCP), движки
  Claude Code / Codex, дисковая память (handoff/progress/qa/rules), мониторы,
  расписания, сервисный бот, RPC-хаб с bearer-токеном, lock единственного сервиса.
