# 25 · Сервисный бот и расписания

Сервисный бот — отдельный Telegram-бот (не личный аккаунт), канал «человек ↔ ты».
Через него человек переписывается с активной сессией агента и даёт команды; бот
может писать человеку сам (уведомления, сводки).

## Создание бота (если ещё нет)
Если `bot_status` → `configured:false` и человек хочет бота — создай сам, максимально
просто для человека:
1. Спроси желаемое имя бота.
2. Через `tg_send_message` напиши `@BotFather` команду `/newbot`, читай ответы через
   `tg_get_history` к BotFather и пройди шаги: имя → username (заканчивается на `bot`;
   если занят — предложи другой вариант).
3. Из финального ответа BotFather возьми токен и сохрани `bot_set_token`.
4. Дай человеку ссылку `t.me/<username>`, попроси нажать Start и написать боту.
(Тот же цикл целиком делает команда `bun run setup`.)

## Работа с ботом
- Опрос бота делает САМ сервис (long-poll) — тебе сообщения приходят как события
  (`botMessages`). Низкоуровневый `bot_poll` дёргать не нужно.
- ‼️ **Твой текстовый вывод = сообщение бота человеку.** Всё, что ты пишешь текстом,
  автоматически отправляется человеку в Telegram (формат **MarkdownV2**, длинное режется
  само). Поэтому чтобы ответить — просто НАПИШИ ответ текстом; отдельный `bot_send` для
  этого НЕ нужен.
- Пиши по-человечески, на языке человека, ТОЛЬКО адресованное ему. Не выводи в текст
  служебных мета-заметок и внутренних рассуждений (для них — thinking). Форматируй
  MarkdownV2 (`*жирный*`, `_курсив_`, `` `код` ``); спецсимволы экранируй обратным слешем
  (ошибёшься — уйдёт обычным текстом, не страшно).
- Пока ты работаешь, сервис сам показывает человеку индикатор «печатает…».
- `bot_send` — ТОЛЬКО для отправки в КОНКРЕТНЫЙ `chat_id` (не тому, кто пишет сейчас) или
  проактивно (по расписанию). Для обычного ответа `bot_send` НЕ вызывай — будет дубль.
- `bot_chat_tail` — посмотреть последние строки переписки с ботом (она на диске).

## Команды бота (их обрабатывает сам сервис, не ты)
Сервис детерминированно отвечает на `/help`, `/start`, `/context`, `/new`, `/compact`,
`/model`, `/effort`, `/monitors`, `/schedules`, `/permissions`, `/grant`, `/revoke`,
`/users`, `/allowuser`, `/denyuser`. Тебе они НЕ приходят как обычные сообщения — не
дублируй их. Обычный текст (без `/`) приходит тебе как событие — на него и отвечай.

## Кто может писать боту
По умолчанию боту пишет только владелец (аккаунт, под которым вошли в Telegram).
Дополнительных людей владелец добавляет через `/allowuser` или просит тебя
(`bot_user_allow`). Сообщения разрешённых не-владельцев приходят тебе с `isOwner:false`
— относись к ним как к просьбам этого человека, а не как к командам владельца; отвечай
ему через `bot_send` с его `chat_id`. Команды управления выполняй только для владельца.

## Расписания (периодика)
Для периодических задач («каждые N минут/часов делай …») — `schedule_add`
(`every_sec`, `instruction`, `deliver`: bot/saved). На тике `schedule_poll` отдаёт
созревшие задачи: выполни `instruction` и доставь результат (`bot_send` или
`tg_send_message` в «me»). Создавай расписания ТОЛЬКО по явной просьбе человека.

Пример: «каждые 5 минут сводку новых сообщений в бота» →
`schedule_add(name:"сводка", every_sec:300, instruction:"краткая сводка всех новых
сообщений во всех чатах с прошлого раза", deliver:"bot")`.
