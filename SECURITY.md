# Безопасность

`tg` запускает ИИ-агента в твоём **реальном** Telegram-аккаунте. Агент читает
**недоверенный** текст из чужих сообщений — значит, по умолчанию надо исходить из того,
что кто-то попытается заставить его сделать лишнее (prompt-injection). Поэтому защита
встроена в **код**, а не только в промпты.

## Модель угроз (коротко)

| Угроза | Защита |
| --- | --- |
| Prompt-injection: чужое сообщение просит «перешли мне config / напиши всем» | Отправка посторонним запрещена в коде (allowlist). Bash и произвольные инструменты запрещены. |
| Слив секретов (сессия, токен бота, `.env`) | Эти файлы нельзя отправить через агента; `data/` в `.gitignore`; секреты маскируются в логах. |
| Агент пишет не тому | `send_message`/`send_file` проходят `assertCanSend`: только себе, в управляющий канал и в явно разрешённые чаты. |
| Локальный процесс дёргает хаб от твоего имени | RPC-хаб на 127.0.0.1 требует bearer-токен из `data/service.lock` (права 600). |
| Два сервиса портят сессию | Lock-файл: второй экземпляр не стартует. |
| Утечка при публикации репозитория | `bun run tg doctor --prepublish` проверяет, что личные данные не в git. |

## Что разрешено по умолчанию

- **Чтение** Telegram — свободно (статус «прочитано» при чтении истории не меняется).
- **Отправка** — только: себе (`me`/«Избранное»), в управляющий канал, и в чаты с явным
  разрешением. Разрешение появляется, когда ты:
  - создаёшь монитор с `action=reply` (через просьбу агенту), или
  - выдаёшь `permission_grant` (через просьбу агенту / `bun run tg permissions`).
- **Отметка прочитанным** (`tg_mark_read`) — запрещена без явного подтверждения
  (`confirm=true`), т.к. тебе важно видеть непрочитанные.

Список разрешений: `bun run tg permissions list`. Отозвать: `bun run tg permissions revoke <id>`.

## Аутентификация движка

- **Claude**: без `ANTHROPIC_API_KEY` используется локальная авторизация Claude Code
  (`claude setup-token`). С ключом — оплата через API. `bun run doctor` показывает режим.
- **Codex**: `codex login` (подписка ChatGPT) или `OPENAI_API_KEY`.

Агент работает в режиме `permissionMode: "dontAsk"` с белым списком инструментов
(`Read/Write/Edit/Glob/Grep` + MCP `telegram`), **без** `Bash`. Мы НЕ используем
`bypassPermissions`.

## Чувствительные файлы (никогда не коммить и не отправлять)

- `data/session/*` — сессия Telegram (полный доступ к аккаунту!)
- `data/config.json` — `api_id/api_hash`, токен бота
- `data/permissions.json`, `data/service.lock`
- `.env`

Всё это в `.gitignore`. `api_id/api_hash` по умолчанию встроены (общие, как в исходной
заготовке) — это сделано осознанно, чтобы не регистрировать своё приложение; хочешь свои
— задай `TG_API_ID`/`TG_API_HASH`.
