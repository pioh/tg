# Безопасность

`tg` запускает ИИ-агента в твоём **реальном** Telegram-аккаунте. Это **личный локальный**
инструмент: агент работает с **полными правами** на твоей машине (как Claude Code без
ограничений — все инструменты, включая Bash), потому что управляешь им только ты.
Запускай его **только на своей машине и со своими данными**.

Главная встроенная защита — **код-гейт на отправку в Telegram**: что бы ни просили
агента в чужих сообщениях (prompt-injection), он не напишет посторонним без твоего явного
разрешения. Это проверяется в коде хаба, а не только в промптах.

## Модель угроз (коротко)

| Угроза | Защита |
| --- | --- |
| Prompt-injection: чужое сообщение просит «напиши всем / ответь за меня» | Отправка посторонним в Telegram запрещена в коде (`assertCanSend` + allowlist). |
| Слив секретов файлом (сессия, токен бота, `.env`) | Эти файлы нельзя отправить через `send_file`; `data/` в `.gitignore`; секреты маскируются в логах. |
| Агент пишет не тому в Telegram | `send_message`/`send_file` проходят `assertCanSend`: только себе, в управляющий канал и в явно разрешённые чаты. |
| Кто может писать боту | По умолчанию только владелец; остальные — только из allowlist (`/allowuser`). |
| Локальный процесс дёргает хаб | RPC-хаб на 127.0.0.1 требует bearer-токен из `data/service.lock` (права 600). |
| Два сервиса портят сессию | Lock-файл: второй экземпляр не стартует. |
| Утечка при публикации репозитория | `bun run tg doctor --prepublish` проверяет, что личные данные не в git. |

> ⚠️ Полные локальные права означают, что агент (и потенциально prompt-injection) может
> выполнять любые локальные действия — читать файлы, запускать команды. Код-гейт защищает
> только Telegram-отправку. Это осознанный выбор для личного инструмента; не запускай его
> на машине с чужими секретами или в недоверенной среде.

## Что разрешено по умолчанию

- **Чтение** Telegram — свободно (статус «прочитано» при чтении истории не меняется).
- **Отправка** — только: себе (`me`/«Избранное»), в управляющий канал, и в чаты с явным
  разрешением. Разрешение появляется, когда ты:
  - создаёшь монитор с `action=reply` (через просьбу агенту), или
  - выдаёшь `permission_grant` (через просьбу агенту / `bun run tg permissions`).
- **Отметка прочитанным** (`tg_mark_read`) — запрещена без явного подтверждения
  (`confirm=true`), т.к. тебе важно видеть непрочитанные.

Список разрешений: `bun run tg permissions list`. Отозвать: `bun run tg permissions revoke <id>`.

## Аутентификация движка

- **Claude**: без `ANTHROPIC_API_KEY` используется локальная авторизация Claude Code
  (`claude setup-token`). С ключом — оплата через API. `bun run doctor` показывает режим.
- **Codex**: `codex login` (подписка ChatGPT) или `OPENAI_API_KEY`.

Агент работает с полными правами: Claude — `permissionMode: "bypassPermissions"` (все
инструменты, включая Bash), Codex — `sandboxMode: "danger-full-access"`, `approvalPolicy:
"never"`. Это сознательно: tg — личный локальный инструмент. Ограничения остаются только
там, где это критично для Telegram — код-гейт отправки (`assertCanSend`).

## Чувствительные файлы (никогда не коммить и не отправлять)

- `data/session/*` — сессия Telegram (полный доступ к аккаунту!)
- `data/config.json` — `api_id/api_hash`, токен бота
- `data/permissions.json`, `data/service.lock`
- `.env`

Всё это в `.gitignore`. `api_id/api_hash` по умолчанию встроены (общие, как в исходной
заготовке) — это сделано осознанно, чтобы не регистрировать своё приложение; хочешь свои
— задай `TG_API_ID`/`TG_API_HASH`.
